The Visitor ID and signal count are derived only from values shown below and are never stored. The Stable device hash uses only hardware/rendering signals that don't change across sessions (GPU, canvas, fonts, screen, CPU) — deliberately excluding IP, timezone, permissions, and other things that vary by network or setting. The UUID is deliberately written to localStorage so you can compare a real stored identifier against a recomputed fingerprint — see the "Persistence" card. IP/location come from a third-party lookup that fires automatically on load, same as any site would see it. Nothing on this page is logged or sent anywhere else.

Deliberately excluded: this report does not collect or display cookies, any persistent machine/device identifier, your MAC address (no browser can read this — it's not a gap, it's a hard OS/network boundary), or OS-version detail from your user-agent string (redacted below on request).

Reload this page a few times: the stored UUID above stays put because it's sitting in localStorage. Clear it here, then reload — it'll be regenerated as a brand-new value, because there's nothing real backing it. The fingerprint hash next to it, meanwhile, will come back identical every time, unprompted, because it's derived from your actual hardware rather than anything stored.
Precise GPS-level location is a different story: it's not silently available. The browser must show you an explicit permission prompt, and nothing is read until you click "Allow." This button demonstrates that consented path — no data leaves this tab.
